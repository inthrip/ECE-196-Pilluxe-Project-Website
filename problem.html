Problem Page

Understanding the Problem

This page explains the project problem, why it matters, and shows existing solutions.

Problem Definition

Clearly define the engineering problem your team is solving. Explain what is happening, who is affected, and what specific limitation or inefficiency exists today. This section should state the problem directly and precisely.

Problem Motivation

Explain why this problem matters. Describe the consequences, user pain points, inefficiencies, safety concerns, cost, or technical shortcomings that make this issue worth solving.

Current Solutions and Limitations

Identify what solutions already exist and explain why they are not enough. This is where you show the specific gap your project addresses.

Existing Solution 1

Add a short description of one existing approach, product, or system.

Existing Solution 2

Add another current solution and explain how it attempts to solve the problem.

Limitations

Explain where current solutions fall short and what gap your project addresses.

What this page should accomplish

  • Define the problem clearly
  • Show why the problem matters
  • Reference current existing solutions
  • Identify the gap your project addresses

Writing Guidance

Avoid generic statements like “this is an important issue.” State the actual failure, constraint, or inconvenience. Use evidence, realistic examples, and direct language.

The problem page should justify why your solution deserves to exist before you start describing the design.
